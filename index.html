What is pair programming?

A method where two programmers work on the same piece of code together at the same.

One developer acts as a 'driver' typing out the code while the other acts as a 'navigator' directing the high level approach.

Pairing is about communication, learning, and a common goal.

How to be a good pair

  • Switching every 5-15 minutes
  • Be flexible when choosing tools
  • Ask "Is my pair engaged?"
  • Ask "Am I engaged?"
  • Think out loud
  • Ask if you aren't sure
  • Acknowledge successes
Define guidelines before pairing:
  • What is the goal or purpose?
  • What tools and/or techniques?
  • How and when to switch roles?

Why Pair Program?
