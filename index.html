What is pair programming?

Pair programming is a method of having two programmers work on the same piece of code together at the same.

One developer acts as a 'driver' typing out the code while the other acts as a 'navigator' directing the high level direction or approach.

Pairing is more about communication, learning, and working towards a common goal.

Define a list of guidelines before pairing:
  • What is the goal or purpose?
  • What are the suggested tools and techniques?
  • How and when to switch roles?

How to be a good pair

  • Switch every 5-15 minutes or good stopping point
  • Be flexible when choosing tools to use
  • Ask yourself, "is my pair engaged?" or "Am I engaged?"
  • Think out loud and ask if you aren't sure
  • Don't blame or make assumptions
  • Be encouraging and recognize successes

Why Pair Program?

Get exposed to someone else's code

Learn new techniques and tools

Prevent bugs and write better code

Get more done in less time

Learn about your pair as teammate

Leave feeling happy and accomplished
